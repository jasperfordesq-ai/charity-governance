Plain-English summary of the work on this branch, suitable as a pull-request
description. 19 commits; full test/build/lint gate green; production dependency
audit reports 0 vulnerabilities.

## Security & dependency fixes
- **`form-data` CVE (high):** a transitive dependency carried a CRLF-injection
  advisory (GHSA-hmw2-7cc7-3qxx); pinned to the patched `4.0.6` without breaking
  the cross-platform (Linux Docker) build.
- **Login user-enumeration (timing):** login now spends the same bcrypt time
  whether or not the email exists, so an attacker can't discover which emails
  have accounts by measuring response time.

## Correctness fixes (backend)
- **Reminders could be silently skipped forever** if a daily scheduler run was
  missed — now they catch up to the most-urgent reached window.
- **Reminder emails showed the due date one day early** on servers behind UTC —
  now formatted in UTC.

## Web app bug fixes (found by a UI correctness review)
- **Forced logout on normal page loads (high):** when several requests refreshed
  the session at once, the backend's single-use-token protection logged the user
  out. Fixed with a single-flight refresh.
- **Accepting a team invite bounced the new user to login** — now loads their
  session before navigating.
- **The "session expiring" warning never appeared / never auto-logged-out** — an
  effect bug; fixed.
- **Expired-trial users saw a generic "check your connection" error** — now a
  clear "manage billing" message.
- **Silent failures** on close/delete/toggle actions now show an error message.
- **"Unsaved changes" reappeared after a successful save** on the org profile —
  fixed.
- **Deep links lost their destination** when redirected to login — now preserved.

## Accessibility fixes
- The deadline complete-toggle and the document upload input now have proper
  accessible names/labels for screen-reader and keyboard users.

## Tests
- Backend: **159 → 234** tests — every service plus route-guard wiring, including
  multi-tenant isolation, billing webhook integrity, auth token flows, and the
  reminder catch-up logic.
- Web: **34 → 38** — the single-flight refresh and subscription-lapse detection
  are now regression-tested.

## Tooling & docs (for launch)
- `npm run setup:production-env` — generates `.env.production` with the random
  secrets filled in.
- `npm run launch:status` — tells you your current launch step.
- New: `README.md`, `docs/LAUNCH-GUIDE.md`, `docs/SECURITY-REVIEW.md`.

## What this branch does NOT do
It does not make the platform "live." Going to production still requires external
steps only the account owner can take — see `docs/LAUNCH-GUIDE.md`: domain, live
Stripe/Supabase/Resend accounts, hosting + TLS, approved legal policies, an
external penetration test, and final sign-offs.
